# Quickstart: Product Details

To begin working on the Product Details feature:

1. Navigate to the `lib/features/coustomer/product_details` directory.
2. The architecture is organized in 2 layers:
   - `data/` (models, datasources, and repositories)
   - `presintation/` (bloc, screens, widgets)
3. **State Management**: Uses `Bloc` (states and events) generated by `freezed`.
4. **Running Generators**:
   Ensure you run the code generator when modifying models or blocs:
   ```bash
   dart run build_runner build -d
   ```
5. Note: "presintation" is intentionally spelled this way to match the project's existing structure unless changed globally.
